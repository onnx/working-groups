## Summary

## ONNX Probabilistic Programming Working Group:
The meeting focused on a working group dedicated to enabling probabilistic programming support within ONNX. The group aims to leverage ONNX architecture to support frameworks such as Stan, PyMC, NumPyro, and others, while potentially enabling the development of an ONNX-native probabilistic programming layer. A key focus area is defining specifications for RNG processes and operators that ensure consistent, deterministic behavior across runtimes and deployment environments, including edge devices with limited compute resources.

## New Team Members and Project Overview:
The group welcomed new participants Grant and Jasmine, who introduced their backgrounds in probabilistic programming. Brian provided an overview of the working group’s goals, including prior discussions with leadership and the broader vision for ONNX probabilistic programming support. The group plans to continue reviewing research on PRNG design and MCMC acceleration while expanding the current RNG specification.

## ONNX Support for Java PPL:
Adam discussed ongoing investigation into ONNX support for deploying a Java-based probabilistic programming language in Python. He highlighted that ONNX currently allows different behaviors between the reference implementation and ONNX Runtime, particularly for RNG operations. Adam presented findings to the ONNX Operators SIG, noting that existing random sampling operators are defined primarily through attributes, which limits flexibility and runtime control.

## ONNX Runtime Random Number Challenges:
The group examined challenges in ONNX Runtime related to RNG behavior. While ONNX aims to represent stateless computation graphs, current RNG implementations do not fully support reproducibility. Seeded execution produces deterministic outputs but lacks runtime flexibility, while unseeded execution introduces variability without control or traceability. These limitations present challenges for probabilistic workflows such as MCMC and reproducible machine learning experiments.

## Random Sampling in LLMs Discussion:
The discussion explored the role of RNG in generative models such as LLMs and diffusion models. The group highlighted how randomness affects token sampling and noise initialization, and how reproducibility is critical for debugging, evaluation, and regenerating outputs. Trade-offs between seeded and unseeded randomness were discussed, including risks of bias and lack of control.

## ONNX PRNG Specification Proposal:
Adam presented a proposal to revise ONNX PRNG specifications to support stateless, splittable random number generators. The proposed approach introduces explicit RNG state handling and aims to ensure deterministic, reproducible, and parallel-safe execution. The group agreed to prioritize this proposal, review missing components, and consolidate feedback into a formal specification.
